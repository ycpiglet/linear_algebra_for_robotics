교재 사용법

깊이 선택, 선수 체크, 복습 큐와 출판 원칙

자유롭게 읽되 연결을 잃지 않기

아무 페이지에서 시작할 수 있지만 아무 배경 없이 모든 깊이를 이해할 수 있다는 뜻은 아닙니다. 페이지 상단에서 현재 목표에 필요한 최소 선수개념을 확인하십시오.

필수 선수의 15초 진단에 답하기 어렵다면 다음 중 하나를 선택합니다.

  • 2분 요약만 보고 계속 진행
  • 완전 복습 페이지로 이동
  • 지금은 건너뛰고 나중에 보강에 저장

우회 학습을 마치면 원래 문단으로 돌아가기가 정확한 절 식별자(anchor)로 복귀시킵니다.

웹 읽기 도구

개념 장의 아래쪽에는 현재 장을 다시 찾기 위한 읽기 도구가 고정됩니다.

  • 핵심: 현재 장의 ‘30초 핵심’으로 바로 이동
  • 즐겨찾기: 장 전체를 저장하고 ‘저장 목록’에서 다시 열기
  • 책갈피: 지금 읽는 절을 제목과 함께 저장
  • 목차: 현재 선택한 읽기 깊이에서 보이는 절만 표시
  • 위치 슬라이더: 현재 절 번호와 읽은 비율을 확인하고 원하는 위치로 이동
  • 저장 위치로: 지난번 읽던 비율로 복귀

키보드에서는 Alt+F가 즐겨찾기, Alt+B가 절 책갈피, Alt+T가 미니 목차를 연다. 읽기 위치는 브라우저의 자동 스크롤이 아니라 휠·터치·키보드·슬라이더처럼 사용자가 실제로 이동했을 때만 저장한다.

전자책(EPUB)은 화면 크기와 글꼴에 따라 페이지가 다시 나뉘므로 웹의 고정 슬라이더를 넣지 않는다. EPUB 전용 뷰어가 제공하는 마지막 위치·책갈피와 이 교재의 장·절 목차를 사용한다. PDF는 실제 페이지 번호와 문서 책갈피를 사용한다.

모든 개념 페이지의 공통 구조

  1. 30초 핵심: 저자가 전달하려는 결론과 공학적 이미지를 먼저 봅니다.
  2. 선수 체크: 이번 깊이에 필수·도움·불필요한 지식을 분리합니다.
  3. 직관: 물리적 이미지, 가장 작은 예, 비유와 그 한계를 설명합니다.
  4. 작동하는 정의: 기호·모양(shape)·단위·좌표계(frame)를 고정합니다.
  5. 전제 장부: 사실, 관례, 모형 가정, 근사, 경험적 해법(heuristic), 필수 조건을 구분합니다.
  6. 유도: 어떤 출발점에서 식이 나오는지 중간 단계를 전개합니다.
  7. 엄밀히 말하면: 명제의 범위를 좁히고 반례와 실패 조건을 확인합니다.
  8. 증명: 필요한 페이지에서 의존 정리와 증명 수준을 명시합니다.
  9. 공학 적용: 센서·제어·로봇 구현에서 식의 역할과 진단법을 연결합니다.
  10. 숨 고르기: 역사·어원·연구자·실패 사례를 출처와 함께 읽습니다.
  11. 회상 점검: 다시 읽지 않고 설명·계산·유도할 수 있는지 확인합니다.

첫 회독에서 모든 층을 읽을 필요는 없지만, 내용 자체를 삭제하거나 숨긴 채 완결되었다고 주장하지는 않습니다. 웹의 깊이 선택은 같은 원고의 표시 범위만 바꿉니다.

읽기 깊이

  • 빠르게 보기: 핵심, 직관, 작은 예제
  • 기본 학습: 정의, 가정, 유도, 응용
  • 전체 보기: 증명, 반례, 역사, 심화까지 모두 표시

PDF와 EPUB에서는 모든 내용이 펼쳐집니다.

필요 없음(not_required)은 “영원히 몰라도 된다”는 뜻이 아니라 현재 목표의 이 깊이에는 진입 조건이 아니다라는 뜻입니다. 예를 들어 PID 직관에 확률과정은 필요하지 않지만 센서 잡음과 확률제어(stochastic control)까지 가면 다시 연결됩니다.

학습 기록 관리

숙달 상태와 선택한 읽기 깊이는 서버로 보내지 않고 현재 브라우저의 localStorage에만 저장합니다. 브라우저나 기기를 바꾸기 전에는 JSON 파일로 내보내십시오.

언어와 표기

설명은 한국어를 기본으로 하며 논문과 교재에서 중요한 영어 표현을 병기합니다. 기호, 단위, 배열 모양(shape), 좌표계(frame)는 처음 나올 때 정의하고 중앙 용어집과 연결합니다.

대화 강의의 반영

표시된 질의응답은 학습 로그에 보존합니다. 가치 있는 질문과 설명은 사실 검증과 편집을 거쳐 FAQ, 반례, 보충 유도 또는 새 문제로 정식 반영합니다.
